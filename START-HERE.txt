📧 EMAIL SETUP (Optional):

The tool works WITHOUT email! But if you want to send reports:

1. Copy the example config:
   cp .env.example .env

2. Edit with your SMTP settings:
   nano .env

3. For Gmail (easiest):
   • Enable 2FA: https://myaccount.google.com/security
   • Get App Password: https://myaccount.google.com/apppasswords
   • Add to .env:
     SMTP_HOST=smtp.gmail.com
     SMTP_PORT=587
     SMTP_USER=your.email@gmail.com
     SMTP_PASS=your-16-char-app-password

4. Restart the server

🎯 HOW TO USE:

1. Start the server:
   npm start
   (or ./start.sh)

2. Open browser:
   http://localhost:3000

3. Upload report:
   • Drag & drop your Allure HTML file
   • Or click to browse

4. View instant analysis:
   • Total tests, pass/fail counts
   • Pass rate with color coding
   • Failures grouped by package

5. (Optional) Email report:
   • Enter @vodafone.com email
   • Click "Send Report via Email"
   • Done!

✅ Only @vodafone.com emails accepted (hard-coded)

✅ No permanent storage

🆘 TROUBLESHOOTING:

Problem: Port already in use
  Solution: kill -9 $(lsof -ti:3000) && npm start

Problem: Dependencies missing
  Solution: npm install

Problem: Email not sending
  Solution: Check .env file (or skip email - it's optional!)

Problem: Can't upload files
  Solution: Make sure it's an .html file under 50MB
